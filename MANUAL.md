## 1. 概要

動物病院の入院患者を**1台のiPad / iPhone / PCブラウザ**から記録・管理できる Webアプリです。

### 設計コンセプト
- 麻酔記録（AnesLog）と同じ「**横軸=時刻、縦軸=項目**」のフローシート形式
- ICU/手術室で看護師が紙チャートに書く感覚をそのままデジタル化
- 一度の入力で複数の病院スタッフがシフトを跨いで情報共有
- iPad横向きでの利用を主眼に設計（縦向きも対応）

### データ保存方式（プロトタイプ）
- 各端末の**ブラウザ内（localStorage）**に保存
- 複数端末間の自動同期は v1 では未実装
- 現段階は**1端末で1病院 / 1看護師**での評価用途を想定

### 公開URL
**https://somnus-vet.github.io/hospital-record/**

## 2. はじめに

### 初回アクセス
1. iPad / iPhone / PCのブラウザ（**Safari推奨**）で上記URLを開く
2. 患者一覧画面が表示されます（初回は空）

### デモデータの取り込み（推奨）
評価用に「ポチ（5歳トイプードル・急性膵炎・1週間入院）」のフル症例データを用意しています。

1. https://raw.githubusercontent.com/somnus-vet/hospital-record/main/demo_data.json をブラウザで開く
2. 表示されたテキストを**全選択コピー**（⌘A → ⌘C / Cmd+A → Cmd+C）
3. アプリ右上の**⬆ボタン**（インポート）をタップ
4. テキスト欄に**ペースト**（⌘V / Cmd+V）
5. 「インポート」ボタン → 「ポチ」が一覧に追加されます

### 新規患者の登録
1. 右上「**＋ 新規**」ボタン
2. 情報タブで基本情報を入力（後述）
3. ⚡記録ボタンで最初のバイタル入力

## 3. iPad / iPhone での使い方

### 推奨環境
- **iPad**: 横向き（Landscape）、Safari、iOS 16以降
- **iPhone**: 縦向き、Safari、iOS 16以降
- **PC**: Chrome / Edge / Safari、画面幅1280px以上推奨

### ホーム画面に追加（ネイティブアプリ風）
1. Safari下部の**📤 共有ボタン**
2. 「**ホーム画面に追加**」
3. ホーム画面のアイコンから起動 → アドレスバー無しの**全画面**で起動

### iPad の主な操作
- **横スワイプ**: 時刻列を左右にスクロール（時刻ヘッダー固定）
- **縦スワイプ**: 項目をスクロール（時刻ヘッダー固定）
- **セルタップ**: 直接編集 or 列ハイライト
- **長押し**: テキスト選択
- **2本指ピンチ**: ブラウザのズーム（要に応じて）

### iPhone の制約
- 画面が小さいため、左ラベル列・時刻列が自動で細くなります
- 申し送りタブの「直近12時間バイタル」が見やすい設計
- 詳細な編集は iPad / PCを推奨

## 4. タブ別機能ガイド

下部のタブから各画面を切り替えます。

### 🔔 申し送り（シフトチェンジ用ダッシュボード）
**患者を開いた最初の画面。看護師が出勤直後に見るべき情報を集約。**
- **🚨 アラート**: 直近バイタル基準外 / 投薬予定超過 / 緊急検査未完了
- **📊 直近12時間のバイタル**: TPR/SpO₂のスパークライン+トレンド↑↓
- **💧 直近24時間のIn/Out**: 食事%/飲水/尿/便/輸液速度の集計
- **⏰ 次の8時間の予定**: 投薬・検査の予定リスト（○分後/○時間後）
- **⏱ 直近8時間の活動**: 全イベント時系列降順（誰が何をしたか）

### 📊 バイタル（縦=項目、横=時刻のフローシート）
- 各時刻列に体温/心拍/呼吸/SpO₂/食事%/飲水/尿/便スコア/担当を記入
- **基準範囲外**は自動的に赤ハイライト
- 「**＋ 時刻**」: 任意の時刻列を追加
- 「**📅 予定追加**」: バイタルチェック予定を一括で組む
- 「**尿 ○**」「**便 ○**」: 現在時刻で即マーク（ラウンド中の最速入力）

### 💧 In/Out（食事・排泄に特化したフローシート）
- バイタルタブと**同じ時刻列を共有**（時刻一致なら統合表示）
- 食事内容のポップアップ選択（i/d缶、k/d缶など14種類）
- 食事量g・飲水mL・尿mL・便スコア1-7（色付）

### 💉 治療
- **輸液**：製剤ごとに1行、各時刻セルに速度mL/hを直接編集
- **薬剤**：薬剤名+用量ごとに1行、セルタップで✓実施トグル
- 投薬予定時刻超過は**赤で点滅**
- 「💊 薬剤一括予定」: **複数薬剤×時刻×日数を一括作成**（後述）
- 「📅 単一予定」: 1薬剤の単発スケジュール
- 「＋ 輸液」: 新規輸液製剤を追加

### 🔬 検査（4状態カンバン）
- **📋 依頼中 / 💉 採取済 / ⏳ 結果待ち / ✅ 結果あり** の4列
- 緊急依頼は🚨マーク+赤帯
- 結果テキストに `↑` `↓` を入れると赤の異常マーク自動表示
- **📷 結果写真添付**: ファイル選択 → 1280px JPEG 82%品質に自動圧縮

### 👨‍👩‍👧 面会
- 日時・面会者（ご主人/奥様等）・対応スタッフ
- **飼い主の様子**: 😊安心 / 😐普通 / 😟不安 / 🤨不信感 / 😠怒り / 😢悲嘆 の6色ボタン
- 面会内容の自動拡張テキスト
- **🎁 頂き物メモ**

### 📝 メモ
- フリーフォーマットの観察メモ・申し送り
- 活動性・疼痛・担当を記録
- 自動拡張テキストでスクロール無しで全文表示

### 📈 グラフ
- TPR/SpO₂を7日間の折れ線で可視化（基準帯付）
- 体重推移
- 便スコア棒グラフ

### 📋 情報
- 患者基本情報（ID/名前/動物種/性別/生年月日→自動年齢計算）
- 主治医・**病名**（上部に白チップで常時表示）
- 診断/主訴・**退院目標**（自動拡張テキスト）
- 基準範囲（アラート判定用）
- ☣ **感染性疾患**：FIV / FeLV / パルボ / SFTS / レプトスピラ / ジアルジア / 疥癬 / ノミ・マダニ / MRSA
- 🎭 **動物の性格・飼い主の対応注意**（**🔒 裏メニュー**は §6 参照）

### ⚙️ 設定
- 表示項目カスタマイズ（BP/CRT/GCS等の追加表示）
- ブラウザ通知（基準外バイタル/投薬予定超過の通知）
- 📄 退院サマリ自動生成
- 🎨 **色凡例**（スタッフ専用）
- データエクスポート/退院アーカイブ/削除

## 5. 主要操作

### ⚡ クイック記録（FAB）
- 画面右下の**大きな⚡ボタン**
- タップ → 4バイタル（体温/心拍/呼吸/SpO₂）+ 担当 + メモ を1画面で同時入力
- 基準外の値は入力中に赤ハイライト
- ラウンド中の**最頻アクション**

### ↶ Undo（取り消し）
- 上部ヘッダーの**↶ボタン**または**⌘Z / Ctrl+Z**
- 30件履歴、患者ごとにリセット
- 全データ操作（バイタル編集/便スコア変更/投薬完了等）に対応

### 💊 薬剤一括予定（治療タブ）
複数の薬剤に共通の投与時刻を一括設定できます。
1. 治療タブ → 「💊 薬剤一括予定」
2. 薬剤リストに**複数薬剤を追加**（名前 / 量 / 単位 / 経路）
3. 開始日・日数（1/2/3/5/7日）を選択
4. 投薬間隔プリセット（**q4h / q6h / q8h / q12h / q24h**）を1タップ
5. 「💊 一括作成」→ N薬剤 × M時刻 × D日 = 全件を1ステップ作成

例：セファゾリン100mg IV + ファモチジン2mg PO + アモキシシリン50mg PO の3薬剤を、q8h × 3日 = **27件を1タップ作成**

### 🟡 列ハイライト（時刻↔値の対応確認）
- バイタル/In・Out/治療タブで**任意のセルをタップ**
- そのセルの**時刻列全体が黄色く強調**
- 縦に長いシートでも「これは14:00のデータ」が一目瞭然
- 同じ列を再タップで解除

## 6. 🔒 スタッフ専用機能（裏メニュー）

> ⚠️ **以下の内容は飼い主様には絶対に見せないでください。**
> 飼い主が iPadを覗いた際に意味のわからない「カラフルなデザイン」として認識される設計です。

### 6.1 タブ下線色エンコード
タブの下線色に、スタッフ用の隠し情報を埋め込んでいます。
飼い主には**ただのカラフルなUI**にしか見えません。

| タブ | エンコード情報 | 色の意味 |
|---|---|---|
| **情報** | 動物の性格 | 灰=普通 / 青=怖がり / 赤=怒りやすい / 緑=人懐っこい |
| **設定** | 飼い主の対応注意 | 緑=普通 / ピンク=協力的 / 赤=クレーマー気質 / 暗赤=攻撃的 / 橙=過保護 / 青=心配性 / 黄=理解度低い |
| **面会** | 最新面会の気分 | 緑=安心 / 灰=普通 / 橙=不安 / 紫=不信感 / 濃赤=怒り / 青=悲嘆 |

**未設定時**: 各タブの下線は薄グレー（記載なし）

### 6.2 設定方法（情報タブで入力）
1. 情報タブを開く
2. ☣ **感染性疾患** カード → 該当疾患をタップ（複数選択可）
3. **動物の性格** カード → 4種から1つ選択
4. **飼い主の対応注意** カード → 7種から1つ選択
5. → 上部のタブ下線色が即座に反映

### 6.3 上部の表示
| 表示 | 意味 | 飼い主に見られて問題か |
|---|---|---|
| ☣ **赤いパルスバッジ** | 感染性疾患 | ✓ 医療情報なのでOK |
| **白チップ（病名）** | 主病名 | ✓ 医療情報なのでOK |
| 🎁 **黄バッジ** | 頂き物の回数 | ✓ OK（数字のみ） |
| 〇〇日目バッジ | 入院日数 | ✓ OK |
| **タブ下線の色** | 上記6.1の隠し情報 | ❌ **デザイン扱い**で通る |

### 6.4 面会気分の記録
1. 面会タブ → 「＋ 面会記録」
2. 飼い主の様子から6色ボタンを1つ選択
3. 保存すると **面会タブの下線**が選択した色に変わる
4. 直近の面会気分が一目で分かる（スタッフのみ）

### 6.5 色凡例の表示
- 設定タブ → 「🎨 凡例を表示」
- スタッフが色を忘れたときの参照用
- **飼い主にこの画面を見せないこと**

### 6.6 退院サマリへの非反映
これらの裏メニュー情報は**退院サマリには一切含まれません**：
- 動物の性格・飼い主の対応注意・面会気分
- 設定タブ → 「📄 サマリを生成」で生成されるテキストには病名・診断・バイタル・治療経過のみ
- → 飼い主にコピー/印刷して渡しても安全

## 7. データ管理・退院サマリ

### エクスポート / インポート
- 右上**⬇ボタン**: 全患者データをJSONダウンロード
- 右上**⬆ボタン**: JSONをペーストしてインポート
- 各患者ごと: 設定タブ → 「この患者をエクスポート」

### 退院サマリ自動生成
- 設定タブ → 「📄 サマリを生成」
- 入院期間 / 診断 / バイタル経過(入→退) / 体重推移 / 食事・便推移 / 治療 / 検査異常 / 最終所見を自動でテキスト化
- 「📋 コピー」でクリップボードへ → 紙カルテに転記、メール送付等
- **裏メニュー情報は含まれません**

### 退院処理
- 設定タブ → 「退院 → アーカイブ」
- 患者一覧から非表示、データは保持

### 完全削除
- 患者一覧で「×」ボタン または 設定タブ → 「削除」
- ⚠️ 取り消し不可

## 8. 既知の制限・注意点

### 現段階で未実装の事項
- **複数端末間の自動同期**: 各端末で独立。AnesLogのようなSheets連携は今後実装予定
- **印刷用PDF出力**: テキストコピーで代用
- **マルチユーザー権限**: 全員が全機能アクセス可（看護師/医師の区別なし）

### iOS Safari の制約
- **ブラウザ通知**: iOS 16.4+のSafariで**ホーム画面に追加した場合のみ**動作
- **localStorage上限**: 5〜10MB（検査写真の枚数に注意）
- **ファイル添付**: カメラロール/カメラから選択（自動圧縮）

### 既知のバグ・未完成箇所
- v1プロトタイプのため、レイアウト微調整が必要な場面あり
- フィードバックを反映して順次改善予定

### バックアップ推奨
- 重要な記録は毎日1回**エクスポート（⬇）→ ファイル保存**を推奨
- ブラウザのデータ削除でアプリ内データが消える可能性あり
